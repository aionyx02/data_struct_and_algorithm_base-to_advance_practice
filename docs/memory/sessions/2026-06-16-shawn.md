## Stage A2 Segment Tree Beats Batch

- Added `A20-segment-tree-beats` (D5) as the twelfth and final Stage A2 problem:
  a Ji Driver (Segment Tree Beats) tree for range `chmin` with range `sum` and
  range `max`. Each node tracks max, strict second max, count of max, and sum;
  `chmin` skips when `x >= max`, applies in `O(1)` when `secondMax < x < max`,
  and otherwise recurses into both children.
- Created `problem.json`, `statement.md`, four fixed test pairs (`01-public`,
  `02-boundary`, `03-beats-second-max-invariant`, `04-hidden-sequence`), one
  correct fixture, and two known-wrong fixtures: `..._no_recurse.cpp` (applies
  the tag on any covered node, skipping the Beats recursion) and
  `..._full_count.cpp` (scales the sum drop by the whole node size instead of
  the max count).
- Added `generateSegmentTreeBeats` (oracle = brute array). The fixed prefix
  pins indices 0 and 1 (always a left-spine node) to `2000` over `1000`, then
  `chmin 0 1 1500` (between second max and max -> exposes the full-count bug) and
  `chmin 0 1 500` (at/below the second max -> exposes the no-recurse bug), so
  both wrong fixtures fail on the first generated case regardless of seed.
  Registered the generator and wired catalog, Judge, and stress CTest entries.

## Stage A3 Disjoint Sparse Table (A22)

- Added `A22-disjoint-sparse-table-sum` (D3): a static Disjoint Sparse Table that
  answers range `sum` in `O(1)`. Unlike a plain Sparse Table (idempotent only),
  it splits each level into segments of length `2^(h+1)` and stores
  suffix/prefix aggregates toward each segment midpoint, so a query combines two
  disjoint halves chosen by `highestBit(left XOR right)` and works for the
  non-idempotent sum.
- Added `generateDisjointSparseTableSum` to `static_range_generators.cpp` and
  registered it. The fixed prefix pins a non-zero middle element, so `sum 0 2`
  (non-power-of-two range) exposes the overlap double-count and `sum 1 1`
  exposes the doubled single element.
- Added `problem.json`, `statement.md`, four fixed test pairs (`01-public`,
  `02-boundary`, `03-disjoint-overlap-invariant`, `04-hidden-sequence`), a
  correct fixture, and two known-wrong fixtures: `..._overlapping_blocks.cpp`
  (reuses an overlapping Sparse Table, double-counting the overlap for sum) and
  `..._missing_single_case.cpp` (omits the `left == right` guard, doubling a
  single element). Wired catalog, Judge, and stress CTest entries.

## Stage A3 Square Root Decomposition (A23)

- Added `A23-sqrt-decomposition-range-sum` (D3): point `set`/`add` and range
  `sum` over blocks of size about `sqrt(n)`, each caching its sum. A query reads
  the two partial boundary blocks directly and adds cached sums for the whole
  blocks between.
- Added `generateSqrtDecompositionRangeSum`: forces at least three blocks
  (`n` in `[9, 30]`), pins a non-zero `a[0]`, and front-loads `sum 1 (n-1)` then
