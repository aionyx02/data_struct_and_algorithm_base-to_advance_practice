## Detailed Notes

- One off-by-one bug was found and fixed during stress validation: the `emitted`
  counters for the A29 and A33 generators started one above the actual pinned-op
  count, so generated inputs declared one more operation than they contained,
  causing the correct fixtures to fail stress. Fixed to 12 (A29) and 10 (A33).
- Each generator pins an invariant prefix that buries a populated/weighted root
  (A29/A31/A32), forces a depth-2 find (A30), runs a multi-union rollback past a
  compressing query (A33), forks old versions before querying them (A34), or
  re-assigns the same start slot (A35), so the matching known-wrong fixtures
  diverge deterministically rather than relying on random luck.
- `A33` adds `std::stack` to `forbidden_symbols` so the undo log must be
  hand-managed; this is the rollback-vs-compression lesson.
- `A34` uses copy-on-write full arrays for the oracle and reference (correct for
  small inputs); the curriculum point about structural sharing is left to the
  solver, with `memory_limit_mb` raised to 128.
