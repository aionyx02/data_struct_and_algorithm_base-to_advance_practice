## Detailed Notes

The template's strict placeholder guard scans every Markdown file under `docs/`, including templates.
Project initialization therefore replaced instructional angle-bracket placeholders in reusable task,
ADR, and session templates. The C++ build commands remain documented as planned commands because the
CMake application skeleton is intentionally outside this initialization task.

The first Judge slice uses direct Windows process creation rather than a command shell. Child
processes run inside a kill-on-close Job Object, and command-line arguments are reconstructed from
the Windows wide-character command line so Traditional Chinese and space-containing paths work
under MinGW.
