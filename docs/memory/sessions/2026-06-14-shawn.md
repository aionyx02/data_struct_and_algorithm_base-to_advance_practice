## Detailed Notes

Graph exercises expose representation invariants directly through matrix rows,
ordered neighbor views, degree counters, edge counts, and reusable edge-table
slots. Shortest paths, MST, topological sorting, and other graph algorithms
remain deferred.

Hashing exercises expose exact table layouts and probe counts so that a
submission cannot pass by implementing set membership with a different hidden
representation. Tombstone state, node-pool slots, load factor, and capacity
changes remain directly observable.

Search-tree exercises expose rotations, parent links, stored heights, balance
factors, node occupancy, colors, and black height. The final Basic Concepts
problems expose exact shift and copy costs while verifying that ADT behavior
survives a representation change.

## Refactor: Split The Stress Generator Monolith

- `src/stress_service.cpp` had grown to ~7,400 lines mixing 73 problem-specific
  brute-force generators with the `StressService` orchestration and a 73-branch
  `if (problemId == ...)` dispatch. Hard to navigate and a two-edit chore per
  new problem.
- Introduced a registry abstraction (`include/judge/case_generator.hpp`):
  `GeneratedCase`, `CaseGenerator`, `CaseGeneratorRegistry`, and a
  `generateCase` that looks up the generator by ID instead of a linear chain.
  `src/case_generator.cpp` assembles the registry from the domain registrars.
- Split the generators into seven self-contained domain translation units under
  `src/generators/` (linear, tree, bst, forest_dsu, graph, hashing, advanced),
  each exposing a `register<Domain>Generators(...)`. Shared helpers
  (`randomInt`, `randomValue`, `appendLine`, `appendIntList`) moved to
  `src/generators/generator_support.hpp` as inline functions.
- `src/stress_service.cpp` shrank to ~166 lines of pure orchestration.
- Adding a problem now touches one domain file plus one registrar line, instead
  of editing the monolith twice.
- Behaviour is unchanged: byte-identical generators, full suite green at
  354/354 after the move. No public contract, security, or dependency change,
  so no ADR required.
