## Detailed Notes

- `A25` metadata forbids the same direct container shortcuts as prior A3
  exercises, with `std::multiset` added because it directly obscures the range
  order-statistic lesson.
- The correct fixture builds node-level sorted arrays and uses
  `std::lower_bound` for `count_lt`, `std::upper_bound` for `count_le`, and a
  binary search over unique values for `kth`.
- The known-wrong fixtures fail fixed and stress tests:
  - `count_le` implemented with exclusive lower-bound semantics.
  - range counting that drops the inclusive right endpoint.
- `A26` metadata forbids the direct container shortcuts used across A3 and adds
  `std::multiset` to keep range order-statistic queries in the intended
  structure.
- The correct A26 fixture compresses values, stores node prefix-left counts, and
  translates inclusive ranges into child sequences for `kth`, `count_le`, and
  `freq`.
- The A26 known-wrong `kth` fixture initially failed fixed tests but passed the
  CTest stress seed because it used the whole-array value midpoint, which random
  tail values could shift. It was changed to a deterministic rank-offset bug so
  the generator's pinned `kth 0 5 5` case catches it reliably.
- The correct A27 fixture compresses values, builds per-level prefix-one arrays,
  records zero-count offsets, and uses half-open matrix ranges for `kth` and
  count queries.
- A27 known-wrong fixtures cover a missing zero-count offset during one-side
  `kth` descent and a missing zero-count offset during count-range translation.
- The correct A28 fixture stores membership in `uint64_t` words with manual
  popcount/trailing-zero helpers, masks partial boundary words for range count,
  and keeps a cached total synchronized across `set`, `clear`, and `flip`.
- A28 known-wrong fixtures cover unmasked boundary-word counting and stale cached
  total maintenance.
